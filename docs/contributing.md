# Contributing

## Prerequisites

- Go 1.21+
- [just](https://github.com/casey/just) (task runner, optional but recommended)
- `age` binary for secrets tests (optional; tests skip gracefully if absent)
- `git` binary for repos tests

## Code Style

- No third-party deps beyond `cobra` and `BurntSushi/toml`
- All errors must be `*errs.DotsError` (or wrapped) with a `Hint` field
- Every user-facing operation must be idempotent
- Generated files get a `# Generated by dots` header
- User-editable files are backed up (`.dots-bak`) before overwrite

## Building and Testing

```sh
just build          # go build -o bin/dots ./cmd/dots
just test           # go test ./...
just fmt            # gofmt -w .
just vet            # go vet ./...

go build ./...      # compile check without producing binary
go test ./... -v    # verbose test output
go test ./internal/shell/... -run TestInsertBlock  # single test
```

## Adding Features

1. Implement in the relevant `internal/` package
2. Write table-driven tests using `t.TempDir()` for filesystem work
3. Wire the command in `cmd/dots/commands.go`
4. Update `docs/` if the change is user-facing
5. Write an ADR (`adr/NNN-title.md`) for significant design decisions
6. Commit with a [Conventional Commits](https://www.conventionalcommits.org/) prefix

## Commit Messages

```
feat: add SSH managed mode        # bumps minor
fix: handle stale symlinks        # bumps patch
feat!: breaking change            # bumps major (minor while < 1.0)
chore: update CI                  # no version bump
docs: update README               # no version bump
test: add e2e scenario            # no version bump
refactor: split cli module        # no version bump
```

Keep the subject line under 70 characters. Use the body for detail when needed.
